# Blipkin Sprite Generation Specifications

## Important Notes for Image Generation
- All sprites should be **true 8-bit pixel art style**
- **64x64 pixels** for consistency
- **Bold black outlines** around all shapes
- **Simple shading** (2-3 colors per element max)
- **Vibrant, saturated colors**
- **Transparent background** (PNG format)
- **Unique creature design** - no resemblance to existing characters

---

## Core Blipkin Design DNA
- **Round/blob-like body** (not angular)
- **Large expressive eyes** (2 circular eyes with highlights)
- **Small limbs or nubs**
- **Growth indicator on head** (sprout → core → glow depending on stage)
- **Primary color palette**: Cyan (#00D9FF), Pink (#FF6EC7), Yellow (#FFD93D)
- **Secondary colors**: Purple (#A855F7), Green (#34D399), Orange (#FB923C)

---

## Stage 1: Baby Blipkin (Hatchling)

### Description
- Tiny round blob body (~40% of canvas)
- Oversized circular eyes (cute proportions)
- Tiny pixel sprout on top of head (3 green pixels)
- Small nub limbs (barely visible)
- Primary color: Cyan with pink cheeks

### Animations Needed
1. **idle-baby**: 2 frames - subtle bounce
2. **happy-baby**: eyes wide, sprout wiggles
3. **hungry-baby**: eyes watery, mouth open
4. **sleep-baby**: eyes closed, tiny zzz particles

**Prompt for IMAGES tab:**
```
8-bit pixel art, 64x64, baby blob creature called Blipkin, round cyan body, huge circular eyes with white highlights, tiny green sprout on head, small nub arms, pink cheek dots, bold black outline, simple shading, vibrant colors, transparent background, idle pose bouncing slightly, cute Tamagotchi style
```

## Generation Instructions

1. Go to the **IMAGES tab** in Vibecode app
2. Copy each prompt above
3. Generate the image
4. Save with naming convention: `blipkin-{stage}-{animation}.png`
   - Example: `blipkin-baby-idle.png`
   - Example: `blipkin-mega-nurturer-idle.png`
5. Upload all generated sprites to the assets folder
